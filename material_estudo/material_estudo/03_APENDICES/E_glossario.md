# 📖 APÊNDICE E — Glossário EN → PT (técnico + business)

## Técnico
| Termo (EN) | PT / Definição |
|------------|----------------|
| **Agent / AI agent** | Agente de IA — modelo que age (lê arquivos, roda comandos, edita código) |
| **CLI** | Command Line Interface — interface de linha de comando (terminal) |
| **IDE** | Ambiente de desenvolvimento (Cursor, VS Code) |
| **Front-end** | Tudo que o usuário vê |
| **Back-end** | Código invisível ao usuário (inclui o banco) |
| **Database** | Banco de dados — armazena/persiste dados para muitos usuários |
| **Postgres** | Banco relacional usado pelo Supabase |
| **RLS** (Row Level Security) | Política de segurança por linha no banco |
| **Anon key / Service Role key** | Chave pública vs. chave privilegiada (nunca exponha a Service Role) |
| **Migration** | Script SQL que altera a estrutura do banco |
| **Framework** | Estrutura pronta (ex.: Next.js sobre React) |
| **One-shotting** | Construir o app inteiro com um único prompt |
| **Refactor** | Reescrever/reorganizar código sem mudar o comportamento |
| **Context window** | Janela de contexto do modelo (tokens) |
| **Context pollution / rot** | Poluição de contexto — informação irrelevante degradando o agente |
| **System prompt** | Instrução base sempre carregada (CLAUDE.md/AGENTS.md) |
| **Skill** | Expertise reutilizável (pasta com SKILL.md), carregada sob demanda |
| **MCP** | Model Context Protocol — conecta ferramentas ao agente (ex.: Playwright) |
| **Hook** | Gatilho que o harness executa (força comportamento, ≠ CLAUDE.md) |
| **Symlink** | Link simbólico — dois arquivos que são o mesmo (CLAUDE.md ↔ AGENTS.md) |
| **Env var / .env** | Variável de ambiente (segredos, API keys) |
| **API** | Interface para serviços conversarem (Stripe, Resend, etc.) |
| **API key** | Chave de acesso a um serviço |
| **Deploy** | Publicar o app na web (Vercel) |
| **Commit / push / main / revert** | Salvar / enviar / branch principal / desfazer (no curso: "fix forward", nunca revert) |
| **Deep research** | Pesquisa profunda em docs (Perplexity/Grok/ChatGPT Pro) |
| **Reasoning effort** | Nível de raciocínio (low/medium/high/extra-high) |
| **Fast mode** | Inferência acelerada (consome plano mais rápido) |
| **One-shot (verbo)** | Resolver de primeira, num prompt |

## Business
| Termo (EN) | PT / Definição |
|------------|----------------|
| **PMF** (Product-Market Fit) | Ajuste produto-mercado — clientes compram mais rápido que você constrói |
| **ICP** / Target Avatar | Perfil de Cliente Ideal |
| **Painkiller vs Vitamin** | Analgésico (essencial) vs vitamina (descartável) |
| **Churn** | Taxa de cancelamento/abandono |
| **DAU** | Daily Active Users — usuários ativos diários |
| **Retention / Return rate** | Retenção / taxa de retorno |
| **Mom Test** | Método de validação sem induzir mentira educada |
| **Cold outreach** | Prospecção fria (DM, email, call) |
| **Starving crowd** | Multidão faminta — mercado com dor + poder de compra |
| **Early evangelist** | Cliente ideal que já montou gambiarra própria |
| **Pivot / Persevere / Scrap** | Mudar ângulo / persistir / abandonar |
| **Kill criteria** | Critérios definidos com antecedência para abandonar a ideia |
| **First principles** | Raciocínio pelos fundamentos, do zero |
| **Extreme ownership** | Assumir total responsabilidade |
| **Delayed gratification** | Gratificação adiada |
| **MVP → "quick & dirty prototype"** | Protótipo rápido e sujo (em vez de MVP) |
| **10x rule** | Precisa ser 10× melhor que o substituto (Thiel) |
| **TAM** | Total Addressable Market — mercado total endereçável |
| **Rule of 100** | 100 ações de outreach/dia (Hormozi) |
| **Hell yes** | Sinal forte de validação ("com certeza!") |
